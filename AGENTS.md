Проект по продаже строительных материалов.
Архитектура:
1. Бэкенд - монолит.
руты:
	h.GET("orders/:userid")
	h.GET("products/:tagid")
	h.POST("login")
	h.POST("register")
	h.POST("regpay")
	h.PUT("productp/:id") <-говно для админа чтобы он смог изменить все поля в бд
	h.PATCH("order/:id")



при этом backend должен быть по clean architecture (repository -> service -> handler)


2. Фронтенд - React + Vite

Корзина хранится в редис, после нажатия кнопки "оплатить" должна отправлять POST api/regpay содержащий в себе order
В окне авторизации(она должна работать при помощи JWT) после нажатия войти должен отправлять на сервер POST api/login, который возвращает user, если у пользователя роль admin, он направляется на админ панель, на которой заказы подсасываются при помощи запроса api/orders и товары подсасываются при помощи запроса api/products, админ в заказах может изменить статус при нажатии изменить отправляет api/order с id заказа и новым статусом, а в товарах он может изменить каждое поле и удалить этот товар при нажатии кнопки изменить отправляется запрос api/productp с id продукта и всеми полями. Если это обычный пользователь нужно его отправлять на страницу каталога. Регистрация после нажатия Зарегистрироваться отправляет POST api/registr, если сервер отправляет, что всё хорошо переносит пользователя на страницу авторизации, иначе отображает ошибку, которую вернул сервер. Личный кабинет подсасывает заказы с сервера при помощи запроса GET api/orders/:userid, сервер возвращает список orders
